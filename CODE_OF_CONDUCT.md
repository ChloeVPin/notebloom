# Code of Conduct

NoteBloom is a small open source project. These guidelines keep collaboration productive.

---

## Expected behavior

- Be direct and constructive in feedback
- Respect that people contribute in their spare time
- Stay on topic — issues and PRs are for the project, not general chat
- Accept that not every feature request or PR will be merged

## Unacceptable behavior

- Personal attacks, harassment, or discriminatory language
- Spam, off-topic promotion, or low-effort noise
- Publishing anyone's private information without consent

## Reporting problems

If you witness or experience a conduct issue, open a [private GitHub security advisory](https://github.com/ChloeVPin/notebloom/security/advisories/new) or contact the maintainer directly through GitHub.

## Scope

This code of conduct applies to all project spaces — issues, pull requests, discussions, and any other venue where NoteBloom is represented.
